### Key Learnings

The AI agent used the guide as the primary context — this is exactly the CLAUDE.md pattern applied to AI-assisted open source work.
- **Active questioning of AI decisions is the right posture:** Mid-session corrections on unnecessary plugin upgrades, removal of CVE dependencies, and GraalVM version selection all required the human to pause and challenge the agent's choices. The agent was broadly correct but made scope-creep errors (upgrading unrelated plugins) that a passive user would miss.
- **Cross-cutting refactors across multiple services are where AI agents provide the highest leverage:** The same pattern applied 3× with decreasing manual effort — session 1 took ~4.5 hours with close oversight, session 3 ran almost on autopilot. This is the compounding return from building institutional knowledge into migration guides.

- **Prompting principles should evolve with real usage:** The identified gap (no domain-specific rules for SSI/DID and Catena-X contexts) confirms that prompting knowledge is built through practice — the plan's requirement to keep CLAUDE.md updated with pitfalls is the right mechanism to capture these as they emerge
